## ğŸ¯ Key Design Decisions

### 1. Multi-Output Regression

**Why**: Predict all parts simultaneously instead of separate models per part
**Benefit**: Learns correlations between parts (e.g., brake pads and rotors)

### 2. Iterative Forecasting

**Why**: Predict day-by-day using previous predictions
**Benefit**: More accurate for longer forecast periods

### 3. Exponential Backoff Retry

**Why**: Handle temporary network/server issues gracefully
**Benefit**: Robust production-ready client

### 4. Firestore Integration

**Why**: Use your existing database structure
**Benefit**: No data migration needed, works with current workflow

### 5. Flask REST API

**Why**: Standard, widely-supported web framework
**Benefit**: Easy to deploy, scale, and integrate
